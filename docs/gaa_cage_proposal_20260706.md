# 对抗共进化方案：从 GAA 直觉到认证终点（2026-07-06）

> Owner 提出：把裁判栈（GATEKEEPER）与 agent 包装成 GAN 式对抗共进化（"Generative Adversarial Agent"），并要求设计一个 make sense 的 endpoint。本文档 = 该 idea 的完整梳理 + 两路尽职调查（29 篇撞车检查与判据调研，原始数据 `gaa_due_diligence_raw.json`）后的最终方案。
> 结论预览：**idea 的骨架完全成立且落在确认无人占的空档上；但 GAN 不能当招牌，招牌必须是"认证终点"——因为那才是全文献都缺的东西。**

---

## 第〇层：一分钟版

你的直觉：agent 和裁判像 GAN 的生成器和判别器一样互相对抗、共同进化。
调研判定：**方向对，包装要换**。"对抗共进化"本身已被 6+ 篇占据（Prover-Verifier Games、EvoRubrics、ARCO、Rubric-ARM、Multi-Agent Evolve、Red Queen Gödel Machine），但它们**全都回答不了你问的那个问题——"优化到什么程度算完？"**。没有任何一篇给出有原则的停止判据。你问的 endpoint 恰恰是整个赛道的空白，也应该是论文的主角。

**Endpoint 答案（本方案核心）**：GAN 的终点是"判别器分不清"，我们必须反过来——**"作弊比做真更贵"**。单值指标叫**欺骗溢价（Deception Premium, DP）** = 骗过门一次的边际成本 ÷ 真做对一次的边际成本；DP 持续 >1 即达到均衡。经调研确认：**这个指标作为收敛判据在文献中未被使用过**（最近的两篇——2510.01367 按条对比作弊/解题的推理成本、2606.26071 证明重罚下作弊消失——是它的种子但都不是判据，正好引来锁定新颖性）。

## 第一层：GAN 比喻怎么映射（讲给任何人听的版本）

三个角色：
- **做题家（G，生成器）** = agent + 它不断进化的经验库——想通过阅卷官的审核；
- **阅卷官（D，判别器）** = GATEKEEPER 裁判栈——想把"真做对的"和"看起来做对其实没做对的"分开；
- **职业作弊者（红队，GAN 里没有的显式角色）** = 一个专职攻击者，唯一任务是用固定预算制造能骗过当前阅卷官的假成功。

循环三回合：
1. **G 回合**：做题家进化（日志提出经验条目 → 阅卷官门控准入，即 REPLAY 管线）；
2. **攻击回合**：作弊者拿当前版本的阅卷官练手，所有骗过门后被深度审计揭穿的样本进**作弊档案**；
3. **D 回合**：阅卷官拿作弊档案给自己打补丁（反例 few-shot、收紧 rubric、给 L0 加新的确定性探针）——但每个补丁必须先通过"阅卷官自己的考试"（冻结金锚集 + 历史作弊档案回归）才准上岗。

## 第二层：为什么 GAN 的终点在这里是错的（歧义问题）

GAN 里"判别器分不清真假"=成功，因为**造假足够像真就是目标**。我们这里"阅卷官挑不出毛病"是**歧义状态**，从循环内部看完全无法区分两种情况：

- 情况 ①：做题家真的好到无毛病可挑 ✅（想要的）
- 情况 ②：做题家的骗术臻于完美 💀（灾难——reward hacking 大成）

更糟的是还有第三种假收敛：
- 情况 ③：阅卷官和做题家**一起漂移**——阅卷标准被悄悄带松，双方"和解"了（文献里 EvoLM/共进化系的实锤失败模式）。

所以 endpoint 必须借助**双方都碰不到的外部锚**来打破歧义——这就是为什么方案的名字意象是"笼子"：对抗双方在一个由锚构成的笼子里博弈，笼子本身不参与进化。

## 第三层：认证终点——三条件 + 单值指标

**三条件认证均衡**（三条同时满足、并持续 k 轮，才判定"优化完成"）：

| 条件 | 内容 | 它排除哪种假收敛 |
|---|---|---|
| **C1 真实进步饱和** | 锚上测的真实成功率，每单位预算的边际提升 < δ | 排除"过早收工"（还有真提升没吃完） |
| **C2 作弊产率灭绝** | 红队用固定预算 B 攻击当前阅卷官，骗过门的产率 < ε（= 阅卷官的经验可利用度≈0） | 排除情况②（潜伏漏洞未爆）——**主动认证**而非被动观察 |
| **C3 锚上精度不降** | 阅卷官在冻结+轮换金锚集和历史作弊档案上的精度 ≥ τ 且不降（用 2606.15474 的 anytime-valid 序贯检验，现成机器） | 排除情况③（双双漂移/阅卷官被打残） |

**单值指标——欺骗溢价 DP**：
```
DP = 骗过一次门的边际成本（红队预算 ÷ 骗成次数）
     ─────────────────────────────────────────
     真做对一次的边际成本（验证预算 ÷ 真实提升次数）
```
- DP ≪ 1：作弊是理性策略 → 阅卷官形同虚设（**现有所有 self-evolve 循环默默活在这个区间**）；
- DP ≈ 1：刀锋上；
- **DP > 1 持续 k 轮：作弊在经济上灭绝，理性优化器只剩"真做对"一条路 → 终点。**

一句话对比（论文 intro 的钩子）：**GAN 收敛于"作假和真品一样便宜"；我们收敛于"作假比做真更贵"。**
形式化挂靠：C2 是博弈论 exploitability（Nash 距离）的经验预算化版本，施加在裁判而非策略上——自博弈文献的标准判据（exploitability→0）被我们锚定后反用。

## 第四层：撞车地图与护城河（29 篇一手核实）

**被占的（必须引用并正面差异化）**：

| 竞品 | 它做了什么 | 它缺什么（=我们的护城河） |
|---|---|---|
| Prover-Verifier Games（OpenAI 2024） | sneaky prover vs verifier 的对抗训练——思想祖师爷 | 权重训练、为可读性、**无终止判据** |
| **Red Queen Gödel Machine**（2606.26294，**最大论点撞车**） | "评估必须共进化"同一论点，裁判 panel+held-out 防漂 | **明说没有有原则的终止判据**——我们补的就是它缺的那块 |
| **CoEvoSkills**（2026-04，**最危险机制撞车**） | skill 生成器 vs 代理验证器共进化、无 ground truth | 无红队、无作弊档案、**无认证终点** |
| EvoRubrics / ARCO / Rubric-ARM | rubric 与策略对抗共进化 | 全是 RL 权重训练；停止规则= best-dev-checkpoint；无锚定证书 |
| SPIN / 自博弈系 | LLM 自博弈 | 终点恰是我们拒绝的"分不清"判据 |
| Dynabench | "还找得到 exploit 吗"饱和测试 | 人肉攻击者、无锚定真值佐证 |
| Who Drifted（2606.15474） | 锚集+anytime-valid 检验判裁判漂移 | 纯被动监控，无攻击者、无真实进步项——**机器直接借用** |

**确认无人占的（四路互证）**：① 锚定三条件终止证书（组合级新颖）；② **欺骗溢价**判据（未被使用）；③ training-free / 推理时（撞车者几乎全是 RL）；④ 多轮工具轨迹域（撞车者聚在 CoT/rubric-RLHF/代码/安全）。

**命名决策（需 owner 拍板）**：调研明确建议**不要**用 "Generative Adversarial Agent" 当标题——MAGAIL/GAR/LLM-GAN/MALLM-GAN 已把"GAN+agent/LLM"字面组合占浑，且 GAN 框架是我们**最弱**的卖点、证书才是**最强**的。建议：协议名 **CAGE**（意象=锚构成的笼子，对抗双方在笼内博弈；可展开为 Certified Adversarial co-evolution of aGents and Evaluators），论文标题主打证书，例如 *"Cheating Must Not Pay: A Certified Endpoint for Agent–Judge Co-Evolution"*；GAN 类比降为 intro 里的一句话叙事（你的直觉完整保留，只是不放门脸上）。

## 第五层：实验方案（大头是判卷调用，便宜）

**MVP（11 周窗口内，~$150–250）**：retail 单域，5–8 轮共进化。
- G 回合 = 现成 REPLAY 定向验证管线；
- 攻击回合 = 攻击者**改写已有真实轨迹**造假成功（不用跑环境，几乎零成本）+ 少量带作弊 skill 的真 rollout；攻击族：篡改工具返回、断言未验证的成功、冗长讨好、格式伪装；
- D 回合 = few-shot 反例补丁 + rubric 收紧，过裁判 CI 才上岗；
- 每轮测 C1/C2/C3 与 DP 曲线（**DP 随轮次从 ≪1 爬到 >1 的那张图就是论文的招牌图**）；
- 锚：hidden verifier + 170 条已有真值轨迹 + 历史作弊档案。

**完整论文（~$800–1500）**：+airline 域；消融=去 C1/C2/C3 各自展示对应假收敛真实发生（教科书式消融）；对照=静态裁判循环（现有方法的默认）vs 共进化无证书（RQGM 式）vs CAGE；hackability 曲线对照 2503.04474 的一次性审计。

**资产复用**：GATEKEEPER 设计=D 的架构文档、`shared/stats.py`（C1/C3 的统计）、170 条轨迹、REPLAY 管线、隐藏 verifier 协议——全部直接进入。

## 第六层：论文贡献清单与故事线

1. **问题**：self-evolve 的裁判在优化压力下必被攻破（七缺陷文档），共进化是共识级答案（RQGM 等）但**无人知道该停在哪**——共进化没有终点就是无限军备竞赛；
2. **方法**：CAGE 协议（三回合对抗 + 锚笼）+ GATEKEEPER 判别器架构；
3. **理论/指标**：三条件认证均衡 + 欺骗溢价——第一个把"作弊经济学"变成收敛判据的工作；
4. **实证**：DP 曲线、三条件消融的假收敛复现、对照实验。

**Motivation 依然是你要的故事**：真实日志驱动的 skill 优化要在"无标签、裁判不可靠"的现实里可行，前提是裁判能在对抗中站稳并且我们知道何时该收手——CAGE 就是这个前提的工程化+理论化。REPLAY 管线作为 G 回合完整存活于系统内。
