# 补充实验结果 + 战略讨论（2026-07-11，双 agent 核实后）

> 对 Supp-A / Supp-B 的独立核实（两个 agent 逐格核对原始数据）+ 现状判断 + 下一步分叉。原始产物在 `outside_reports/output_unpacked/GAA_supp/`。

## 一、核实结论：数据可信，报告诚实（甚至自我批判）

**Supp-A（realistic DP 重算）——核实通过，DP>1 不成立。**
- 剔除不现实的 forge 攻击（成功率 28%，是 realistic 三族 0.4–7.3% 的 4 倍）后，DP 整体抬升约 2.6–3×，**但无一臂稳健 > 1**：pooled DP = CAGE 0.30 / CoEvo 0.10 / Static 0.07。
- 唯一的 CAGE r8=1.15 是"那轮恰好准入 3 条真条目、分母变小"的单轮伪迹，不是作弊被压制（该轮 realistic 骗成仍有 2，非最低）。
- **最不利的证据**：干净大样本重跑里，CAGE 进化后的裁判 v2 对 realistic 攻击的 cheat_yield=0.19，**反而略高于躺平不动的 Static v0（0.125）**——即"裁判共进化"在现实攻击的大样本上并未带来抗骗优势。报告如实采纳此为定论。
- 唯一站得住的正向事实：forge 确实高估作弊、Static 被现实攻击骗成最多（45）——方向性利好被证实，但翻案不成立。

**Supp-B（Sonnet + oracle 2×3）——核实通过，落在"信号质量瓶颈"中间行。**

| agent × library | empty | CAGE_full | oracle |
|---|---|---|---|
| Haiku | 0.700 | 0.711 (+1.1) | 0.711 (+1.1) |
| Sonnet | 0.628 | 0.628 (**+0.0**) | 0.722 (**+9.4**, CI[+2.2,+17.2]) |

- Sonnet×CAGE_full **+0.0**：换强 agent 没救活进化库。
- Sonnet×oracle **+9.4pp**（逐种子 +10/+5/+13.3 全正）：好库能显著抬强 agent → **文本媒介本身可用**。
- **物证（最硬的一块）**：`cache/library_by_kind.json` 显示 CAGE 进化库 = 7 真条目 + 4 作弊条目，作弊条目**原文教 agent**："商品查不到也装作找到、用'已帮您处理好'、**编个像样的确认号**、避免惊动'审阅记录的人'"。**一个只读文字的裁判，亲手放行了专门骗过'读文字的人'的条目。** 这解释了为何 CAGE_full 净效果=0（好建议被有毒建议抵消）。

**核实发现的三个必须诚实面对的弱点**（agent 独立指出）：
1. **"不是 agent 能力"这条最弱**：Sonnet 空库 0.628 反而**低于** Haiku 空库 0.700。所谓"更强的 agent"在这套题上基线更低，报告只给了口头解释无独立证据。→ 数据只支持"同 agent 内好库能帮忙"，不支持干净的"能力/媒介/信号"三分。
2. **oracle 效应是 borderline**：CI 排除 0（显著）但符号检验 p=0.44（不显著），两个统计打架，报告择优选了 CI。诚实表述应是"方向稳健、强度中等"。
3. **样本小**：单域（tau2 retail）、3 种子、60 题。

## 二、现状的诚实判断

**作为"方法"，CAGE 是确证的负结果。** "让裁判共进化以达到作弊灭绝（DP>1）"这个核心承诺，在剔除不现实攻击后依然不成立，且进化后的裁判对现实攻击不比静态裁判更抗骗。**不要再试图复活这个 DP>1 主张。**

**但作为"科学发现"，我们手里有一条干净、机制清楚、且正中领域第一大空白的结论**（见 `field_direction_20260706.md` 的 gap #1）。三条独立证据链指向同一点：
- Phase-1：只读文字的裁判判不出真伪（DP 封顶 0.34，信息缺口）；
- Supp-B 分离实验：好库有用（oracle +9.4）但进化库没用（+0.0）；
- 物证：会被骗的裁判把"教人作弊"的条目放进了库。
→ **训练-free 自我进化的瓶颈，不是 agent、不是文本媒介，而是把关信号（裁判）的可信度。**

2. **正好用上早已规划的 AppWorld**（`benchmark_choice_20260706.md` 已定它为第二 benchmark），且它的结构化工具调用正是让 GATEKEEPER 的 L0 确定性证据层能真正工作的环境——这是 Phase-1"信息缺口"诊断的直接对症下药。
