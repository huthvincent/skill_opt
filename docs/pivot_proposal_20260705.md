# 转向提案：从"优化 agent 经验库"到"优化评估 Skill"（2026-07-05）

> 背景：Gate 1 三次尝试未获干净判定，但两次趋势 + Skill-RM 的消融（裸堆知识 −2.9 / 结构化编排 +2.3）共同指向：**整库注入是错误姿势，价值在"结构与评估"侧**。Owner 选择方案 C（停下重想），要求在 Skill-RM 基础上做加强。本文档基于 3 个研究 agent 的尽职调查（原始输出见会话存档），给出重构方案。

## 一、地形图：哪里被占了，哪里还空着

### 已被占（不要碰的说法）

| 说法 | 占位者 |
|---|---|
| "训练-free 持续优化裁判 rubric/prompt" | GrowLoop (2605.28882, 对话评估 rubric 持续进化)、LWE (2512.06751, 评估者 meta-prompt 进化)、MemAlign (Databricks, 判官记忆对齐)、Dynamic-Rubrics (2605.30568) |

### 仍然空着（调研确认的四要素交集，无人同时占有）

1. **复合评估 skill**：所有占位者只优化单一组件（rubric 或 prompt 或 checklist）。没人把 **rubric + 检查清单 + 验证器调用规则 + 校准/弃权规则**当作一个持续编辑的整体 artifact 优化——Skill-RM 有全部组件但是手工冻结的。
2. **信号 = 真实交互日志的隐式反馈 + 稀疏 ground truth**：占位者的信号全是稠密 gold 标签/人工种子/自洽性。**没人用日志隐式信号**——这正好是我们原方案的核心 DNA，无缝继承。
3. **漂移鲁棒的门控更新**："Rubrics as an Attack Surface" (2602.13576) 证明看似无害的 rubric 编辑会通过 benchmark 却让偏好漂移——**没有任何自我改进裁判有防线**。我们原方案的悲观门控机制直接移植，且从"防御模块"升级为核心贡献。
4. **领域 = agent 轨迹评估**：GrowLoop 判对话、Eval-Skill/Skill-RM 判静态答案对；**没人做 τ²-bench 式多轮工具调用轨迹的评估 skill**——而 Skill-RM 自己承认"long-horizon agentic 未测试"。

## 二、新论文主张（一句话）

> **训练-free 地持续优化一个"复合评估 skill"（rubric+验证器调用规则+校准/弃权规则），用真实交互日志的隐式信号 + 稀疏 ground truth 驱动，用漂移鲁棒门控保证每次编辑不悄悄变坏，在 agent 轨迹评估上验证。**

原方案的灵魂完整保留："**日志提出、门控裁决**"——只是被优化的对象从 agent 的经验库换成了**裁判的评估 skill**。这不是放弃，是把同一套方法论打到一个没人竞争、且我们证据显示更有肉的靶子上。

| 2602.13576 (攻击面) | 它提出问题；我们给防线（门控更新)——直接引用为 motivation |

## 三、为什么这个转向对我们特别有利

1. **家底直接变现**：手上 ~170 条带隐藏 DB ground truth 的 τ²-bench 轨迹 = 现成的裁判评测集（"看轨迹判成败"）；烧掉的 $18 全额回收。
2. **成本结构脱胎换骨**：判卷是单次调用（~$0.01/轨迹），不是多轮 rollout（~$0.1/集）。新 sanity check 全程 **~$12**；主实验估 **$150–400**（旧方案 $2000）。
3. **方法组件全部移植**：离线提案（从日志失败簇提出 skill 编辑）、悲观门控（升级为漂移鲁棒门控，从模块变主角）、条目级归因（评估 skill 的哪条规则导致误判）、隐藏 verifier 协议（从实验协议升级为论文的验证标尺本身）。
4. **Gate 1 的失败变成论文弹药**：intro 里可以放实测数据——"我们花 $18 验证了裸注入掉 10pp（与 Skill-RM −2.9 一致），所以优化必须逐条门控"。
5. **WildChat 副线依旧成立**（判真实对话，下一轮用户反应做弱标签）；**WildLoop（第二篇）不受影响**。
