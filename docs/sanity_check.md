# REPLAY Sanity Check —— $100 以内验证"这篇论文到底成不成立"

> 目的：在投入 ~$2000 主实验之前，用 **$100 以内**回答一个问题——**REPLAY 的核心机制是真管用、值得写，还是自我感觉良好？**
> 设计原则：**先证伪，且要便宜地证伪**。下面是一串"门"（Gate），按"失败了能最快毙掉论文"的顺序排。**任何一道门没过，就当场停，把后面的钱省下来，回来找我商量。**
> 成本锚点：E0.1 实测 haiku 全角色 **$0.094/集**（run `smoke_tau2_20260705_110319`）。全程用 haiku，不用贵模型。
> 状态：**本文档是计划，尚未执行**（owner 要求暂不花钱）。owner 批准后，按门顺序逐个跑，每过一门更新一次结论再决定要不要跑下一门。

---

## 核心手法：用"藏起来的标准答案"当尺子

τ²-bench 每道题**本身带标准答案**（数据库状态对不对，即 hidden verifier）。我们方法在真实场景里是**看不到**标准答案的（只有 AI 裁判和用户的模糊信号）。

Sanity check 的诀窍：**让我们的方法照常看不到标准答案，但我们（评估方）偷偷拿标准答案当尺子**，去量：
- 有没有提升空间？（Gate 1）
- 我们那个便宜的 AI 裁判信号，到底准不准？（Gate 2）
- 便宜信号驱动的优化，真能提升真实成功率吗？（Gate 3）
- "只信裁判"会不会把说明书带偏？（Gate 4）

这样每一分钱都花在"拿真相验证我们的近似信号"上，而不是盲目跑。

---

## Gate 1 —— 天花板检查：好说明书到底有没有用？（~$12）

**要证伪的前提**：如果给 agent 一份**精心写好的经验库**，它的成功率根本不涨，那"优化经验库"这件事就毫无意义，**整篇论文当场死亡**。这是最该先做、最便宜、最致命的一道门。

**怎么做**：
1. retail 域取 ~30 道题，haiku agent，**不带任何经验库**跑一遍 → 记基线成功率（可复用 E0.1 的部分结果）。
2. 写一份**强经验库**（这一步允许"作弊"：可以让一个强模型**看着标准答案和失败原因**去总结经验条目——因为这里只是测"天花板有多高"，不是测我们的方法）。
3. 同样 30 道题，带这份强经验库再跑一遍。

**成本**：30 题 × 2 条件 × 2 次重复 ≈ 120 集 × $0.094 ≈ **$12**。

**判定**：
- ✅ **过**（强经验库带来 **≥ +10 个百分点**）→ 有提升空间，游戏继续。
- ❌ **不过**（几乎没涨或倒退）→ **停**。要么 retail 域对经验库不敏感（换 airline/telecom 再测一次，仍不过就说明 benchmark 选错了），要么整个"优化经验库"路线在这类任务上无效，需要重新想问题设定。

## Gate 2 —— 便宜信号检查：AI 裁判看得出好坏吗？（~$6）

**要证伪的前提**：我们方法的"便宜一半"是从真实日志里，用 **AI 裁判 + 隐式信号**（看不到标准答案）判断"agent 在哪翻车了"。如果这个便宜信号跟真相**几乎不相关**，那"从日志提出候选"这半就是建在沙子上。

**怎么做**：
1. 用弱 agent（haiku、无经验库）生成 ~50 个"日志 session"。
2. 每个 session 算两个分：
   - (a) **hidden verifier**（标准答案，真相，我们偷偷看）；
   - (b) **AI 裁判 + 隐式信号估计**（看不到标准答案，模拟真实场景）。
3. 看 (b) 能多大程度上区分开 (a) 里的成功/失败。

**成本**：50 session 生成 ≈ $5，裁判调用 ≈ $1，合计 **~$6**。

**判定**：
- ✅ **过**（裁判区分成功/失败的 **AUC ≥ 0.7**）→ 便宜信号可用，可以拿它去提出候选。
- ⚠️ **勉强**（AUC 0.6–0.7）→ 信号弱但有救：说明"悲观门控 + 现场裁决"这后半更重要了（因为提出的候选更脏），继续但心里有数。
- ❌ **不过**（AUC ≈ 0.5，等于瞎猜）→ **停**。要么换更强的裁判提示/多裁判投票再测，仍不行则"从日志提出"这半不成立，方法要退化成纯在线（那就失去和别人的核心区别了，需要重新定位）。

---

## Gate 3 —— 主张检查：日志+现场裁决，真能打赢纯在线吗？（~$45）

**要证伪的前提**：这是论文的**核心主张**——"日志提出 + 现场裁决"在**同样的现场练习预算**下，比"把预算全花在现场练习"（纯在线，别人的套路）更划算。这道门直接决定论文成不成立。做一个**最小可信版本**：1 个域、~20 道评测题、固定一个很小的 rollout 预算。

**怎么做**：三条腿，**给一样的现场练习预算**：
- **(a) 基线**：不优化（复用 Gate 1 的基线）。
- **(b) 纯在线**（模拟 Training-Free GRPO 套路）：把全部预算花在"现场跑题→反思→改说明书"。
- **(c) REPLAY-mini**：候选条目来自 Gate 2 已生成的日志（**免费复用**），同样的预算**只花在现场核实这些候选**。

三者最后都在同一批 held-out 题上用 hidden verifier 评分。

**成本**：优化各 ~100 集 + 评测各 20 题 × 2 次，三腿合计 ~280 集 × $0.094 ≈ $26，加 optimizer 反思/改写的额外 LLM 调用 ~$5，加重试余量 → **~$45**。

**判定**：
- ✅ **过**（**(c) ≥ (b)，且两者都明显 > (a)**；理想是 (c) 更高，或 (c) 用更少 rollout 追平 (b)）→ **绿灯！** 核心主张在小规模成立，可以理直气壮申请 ~$2000 主实验预算把它做扎实、做多域多种子。
- ⚠️ **平手**（(c) ≈ (b)）→ 黄灯：说明"日志"至少不拖后腿、还省了它自己那部分成本；靠"样本效率"叙事仍可发，但需要预算曲线（E1.2）把优势坐实。
- ❌ **不过**（(c) < (b)）→ **停**。日志非但没帮忙还添乱。回来一起看是 replay 探针没起作用、还是候选质量太差，决定是修还是转型（比如转成"judge 鲁棒的纯在线优化器"，即主打 Gate 4 那条线）。

---

## Gate 4 —— 加分项：抓一张"只信裁判会翻车"的招牌图（~$15，预算够才做）

**要展示的现象**：只用 AI 裁判去优化（不加真实日志锚、不加悲观门控），会出现"**裁判打分蹭蹭涨，但真实成功率不涨甚至跌**"的背离——这正是我们悲观门控要治的病。这张图是论文的**招牌卖点**之一。

**怎么做**：跑一个"只信裁判"的优化，同时用 hidden verifier 盯着，画出两条曲线（裁判分 vs 真实分）随优化轮次的背离；再叠加 REPLAY 的悲观门控版本，展示背离被收窄。

**成本**：~100 集 ≈ **$10–15**。

**判定**：这门不是 go/no-go（不毙论文），是"锦上添花"。出现背离 → 强证据，直接进论文 intro 的招牌图；没出现 → 说明这个 benchmark 上裁判还挺老实，把这个卖点降级或换域再试，不影响主线。

## 预算总账

| 门 | 内容 | 成本 | 性质 |
|---|---|---|---|
| Gate 1 | 好说明书有没有用（天花板） | ~$12 | **毙论文**（前提） |
| Gate 2 | AI 裁判信号准不准 | ~$6 | **毙半个方法**（离线半） |
| Gate 3 | 日志+裁决 打不打得过纯在线 | ~$45 | **毙论文**（核心主张） |
| Gate 4 | 抓"只信裁判会翻车"招牌图 | ~$15 | 加分（可延后） |
| 重试/调试余量 ×1.2 | | ~$16 | |
| **合计** | | **~$94** | 卡在 $100 内 |

**执行纪律**：
1. 严格按 Gate 1 → 2 → 3 → 4 顺序，**过一门再花下一门的钱**。
2. 每门跑完，把结论写进 `../REPLAY/LEDGER.md` 和这份文档，再回来找 owner 决定要不要继续。
3. Gate 1 或 Gate 3 没过 = 红灯，**立即停止、不跑后面**，把省下的钱和发现一起汇报。
4. 全程 haiku；每个脚本带 `--max-cost-usd` 熔断（CONSTITUTION §6）。

**这份 sanity check 通过意味着什么**：Gate 1+2+3 全绿 = 我们有确凿的小规模证据证明"真实日志能便宜地帮上忙、且核心主张成立"，这时候申请 ~$2000 主实验预算就是有底气的投资，而不是赌博。三道门任何一道红灯，我们都只花了 ≤$63 就知道该转向了——这就是"便宜地证伪"的价值。
