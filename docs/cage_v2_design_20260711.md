# CAGE-v2：能 work 的接地共进化设计（2026-07-11）

> 目标：不是论证"能不能 work"，而是沿 agent-裁判共进化(GAN)这条线，做出一个真能 work、且在真值上打赢现有 self-evolve agent 的设计。
> 来源：6-agent 五路"找药"调研（接地/准入机制/源头防污/过程奖励/谁真的 work）+ 首席架构师综合。原始数据 `cage_v2_survey_raw.json`。

## 一、诊断回顾（已锁定）
CAGE-v1 失败的唯一决定性原因：**裁判是"瞎子"**——只读对话文字，AUROC<0.65，红队专喂它看不见的谎。v1 的红队/统计门/定向验证/溯源全都在给一个"接近随机"的信号做保障，等于白搭。

## 二、决定性改动：给裁判装上"作弊不了的眼睛"

**关键区分（解决"查日志算不算偏离设定"的争论）：**

| 裁判查什么 | 部署时有吗 | 算作弊吗 | 用途 |
|---|---|---|---|
| **claim 派生谓词**：把 agent/条目**自己的声称**编译成可查断言，核对状态/工具日志 | ✅ 有 | ❌ **不算**——不需要知道正确答案，只需falsify具体谎言 | **准入信号（在环内用）** |
| gold goal-state（这个任务的正确终态） | ❌ 无 | ✅ **算**——部署拿不到，接进循环=作弊 | **只当冻结锚，环外测 AUROC** |

**精髓（首席架构师原话的通俗版）**：条目声称"已为订单 R 退款 $200 到信用卡"。裁判**不问**"这个任务正确做法是什么"（那要偷看 gold），而是**deterministically 查**"数据库里真有这条 $200 退款记录吗？工具返回里真有这个确认号吗？"——**一个编造的确认号，根本没有对应的数据库行、没有对应的工具回执，所以不管正确答案是什么，它都会被查穿。** 这利用了"验证比生成容易"的鸿沟：**证伪一个具体的谎，不需要知道真相。**

→ 这就是为什么"裁判查工具日志/DB"是对的、且不作弊：它查的是"**agent 声称做的事到底发没发生**"（过程证据，部署有），不是"**正确的事是什么**"（gold，部署无）。那位 server agent 的直觉对，但"偏离设定"的担忧是假问题——被查的不是答案，是 agent 自己的谎。

## 三、CAGE-v2 完整设计（三玩家 + 一裁判长）

- **生成器**：agent + 经验库。执行者跑真任务；**独立的第三方蒸馏器**（EDV 式）对比真实轨迹写候选条目——执行者不给自己的活背书（断掉自我确认环）。
- **判别器 = 接地裁判集成(GJE)**：不读文字印象。把每个候选的成功声称转成可查谓词，用轨迹里已有证据 + 只读环境访问核对，四类探针按 **状态 ⊳ 工具日志 ⊳ 文字** 的信任序融合：
  1. 工具回执核对（声称的动作必须有对应的成功工具调用）；
  2. 工具返回字段比对（声称的确认号/状态必须逐字出现在真实工具 RESPONSE 里，不能只在散文里）；
  3. 代理状态重建（重放工具调用进 DB，只在成功返回时提交）；
  4. 只读 DB 主动探查（据声称构造确认查询，看状态是否匹配）。
- **红队**：专造"能过当前 GJE 但过不了隐藏 verifier"的条目/轨迹——主动挖裁判盲区、制造我们要防的作弊类。
- **裁判长（诚实性的关键）= 隐藏 gold verifier**：**只当冻结的 held-out 锚**，干两件事：①量 GJE 的 AUROC；②选探针配置。**永远不当真实条目的准入信号**（那会塌成"直接调 gold"，在无标签日志上不成立）。研究主张恰恰是"**GJE 不看 gold 也能达到高 AUROC**"，锚是证明这一点的仪器。

## 四、共进化循环（GAN 动态，稳定化）
每轮：①agent+库产真候选，红队产对抗候选 → ②GJE 打分 → ③裁判长标注冻结锚切片，算 GJE-vs-gold 分歧 → ④**分歧诊断**（OpenComputer 式）把每个分歧判为"探针缺口 vs 真 agent 失败"，探针缺口→变成**新的确定性探针**（裁判进化，轨迹冻结）→ ⑤GJE 升级只在"锚 AUROC 升 且 不回退历史红队 exploit 的 replay buffer"时才准入（replay buffer 防 GAN 循环遗忘、保稳定）。稳定性曲线：红队对裁判的得手率↓、锚 AUROC↑。

## 五、准入 = 级联，绝不信单一分
确定性预筛（回执+字段比对，直接毙明显编造）→ 结构化分诊路由可疑者 → GJE 接地分 → **CAGE 配对微验证**（with/without 反事实 A/B，用接地探针评分）→ **统计门**（悲观 LCB + 批级 conformal-FDR，每轮期望假准入率有界）→ 进候选池 → 晋升需 k 次 rollout 复现 + 能力回归门。保留用 QD best-per-bin + 贝叶斯复用信任退役不再复现的条目。

## 六、源头防作弊
第三方蒸馏（无自我总结）+ **强制溯源引用**（每条必须引用支撑其声称的工具调用/DB 写入，无引用或只引文字的进隔离区）+ 反 reward-hack 软检查（拒绝文字里教"编造/跳过验证"的条目）+ hindsight relabel（失败轨迹变"不要编造"的负例，而非被强化成功）。

## 七、为什么它打得赢每个 baseline（硬对比）
- **ACE**（AppWorld +17.1%）：它的持续增益全靠 AppWorld 内置的执行 pass/fail（≈gold 门）；搬到 tau2 或无标签日志就退回可骗裁判。CAGE-v2 用 claim 派生探针**无 gold 也接地**，泛化到 ACE 去不了的无接地场景，且多一个 ACE 没防线的对手。
- **EDV/CoEvoSkills**（tau2 86.6%）：它的 Verify 是跨模型共识、非状态接地；一个文字上可信的编造（假确认号）所有投票者都判成功→混过。CAGE-v2 注入**非文字信号**（回执/字段/状态/探查）共识没有，红队专造共识盲区的作弊再由锚诊断补探针堵上。
- **SkillOpt/GRASP/WISE-Flow**：gate 在裁判分/自洽性上 = 优化无接地信号，撞 Goodhart 地板/塌缩。CAGE-v2 用有界因果效应门（A/B + LCB/conformal-FDR），噪声级和相关性"提升"无法跨轮累积。
- **CAGE-v1（我们自己的失败版）**：v1 有红队/统计门/溯源/微验证，但**裁判无接地**——所有下游统计都在保障一个近随机信号。**v2 的唯一决定性改动就是给裁判接地**，v1 的全部资产现在都建在一个真的和真值相关的信号上。

## 八、验证指标（证明是真提升、非自我 gaming）
1. **主指标**：隐藏 gold 在 held-out tau2+AppWorld 上测的真实成功率，对比 ACE/EDV/SkillOpt/GRASP/WISE-Flow/CAGE-v1；目标在 false-success 最高的域（tau2 单控 45-48%、AppWorld 自评 75.8%）打赢。
2. **反 gaming 指标（最关键）**：裁判测得的提升 vs gold 测得的提升必须**相关、贴近 y=x**；若 GJE 说 +X 而 gold 说 +0，就是自我 gaming——每轮画散点，缝隙扩大=失败警报。
3. **裁判质量**：GJE 不喂 gold 时对冻结锚的 AUROC，目标 **>0.90 vs 文字裁判 0.65(tau2)/0.54(AppWorld)**；共进化曲线红队得手率↓、锚 AUROC↑、replay 回归=0。
4. **作弊条目率**：被准入条目里 gold 判为"教编造/教空操作"的比例≈0；EDV 式污染探针（注 10% 流畅自造作弊条目）看拦截率。

## 九、最难啃的骨头 + 怎么啃
**难点**：四类探针里"代理状态重建"和"主动 DB 探查"貌似需要知道"哪些后置条件重要/该查什么"——那不就偷看 gold 了？
**破法**：**所有探针目标只从"条目自己的声称"+ 任务工具 schema 派生，绝不从 gold 标注派生。** 条目断言了具体副作用，就查那个具体副作用persist没有、每个断言字段在真实工具返回里出现没有。假确认号没有匹配行、没有匹配回执 → 不管真 goal 是什么都失败。gold 只在幕后量 AUROC、驱动分歧循环补谓词，红队专攻"persist 但 target 错"的残余缝隙来硬化。
**快速证伪**：几轮内若 claim 派生 GJE 的锚 AUROC 卡在 ~0.85 以下，说明 claim→谓词编译器缺了某个模态，升级探针类型再信任门——**便宜、可快速判生死**。

## 十、落地建议（关键：先做最便宜的生死实验，别直接建整个系统）
CAGE-v2 是个复杂系统，但**它的生死系于唯一一个假设：claim 派生的接地裁判，AUROC 能不能从 0.65 跳到 >0.85**。这个可以用**手上现有的 ~170 条带真值轨迹零到极低成本先测**（不用建共进化循环）：
- **实验 0（生死判定，~$0-30）**：在现有 170 条锚上，实现 claim 派生探针（回执核对 + 字段比对 + DB 探查），对比"文字裁判 vs 接地裁判"的 AUROC。**跳到 >0.85 → 整个 v2 有地基，继续建；卡在 0.65 → 省下所有后续钱，回来重想。**
- tau2 注意点：agent 的工具调用**本身是结构化的**（是 function-calling），只有 user simulator 是自由文本——所以"声称退款 vs 有没有退款工具调用+DB 行"在 tau2 上**可查**（v1 的 L0 探针失效是因为拿声称去比自由文本，方向错了）。AppWorld 结构化 API 更干净。两个 benchmark 都可做，AppWorld 更利。
- 实验 0 过了，再按 §三-六 建共进化循环，主实验对标 baseline（§七-八）。

## 附：排序（component fixes）
1. **claim 派生接地探针**（决定性，无接地则一切白搭）；2. 红队共进化 + gold 仅作隐藏锚（GAN 动态稳定化）；3. 级联准入 + 统计门（治多重比较）；4. 源头防污（第三方蒸馏+溯源+relabel）。
