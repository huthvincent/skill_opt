# Self-Evolve 场景的裁判设计：文献缺陷分析与改进设计（2026-07-06）

> 基于 4 个 research agent 的一手调研（63 篇论文/系统：self-evolve 循环内裁判 17 篇、通用 LLM 裁判 19 篇、"裁判在优化循环里如何坏掉"15 篇、工程级产品 12 个），原始数据见 `judge_survey_raw.json`。
> 问题设定：裁判是 training-free self-evolve 循环（skill/经验库/prompt 优化）的 reward 信号；对象是多轮工具调用轨迹（τ²-bench 式）；优化期无 ground truth；每轮数千次裁判调用；**优化器对裁判施加 Goodhart 压力**。

---

## 一、现状：self-evolve 论文的裁判都是怎么做的（以及它们自己承认什么）

| 方法 | 裁判设计 | 论文自认/实测的弱点 |
|---|---|---|
| ACE | 执行反馈 + LLM Reflector 自读成败 | 附录自认"反馈质量差时 context 变噪声甚至有害" |
| GEPA | ground-truth 任务分数 + 反思 | 大部分 rollout 花在验证集选择（winner's curse）；无标签场景下 μ 变成 LLM judge 但从未压力测试 |
| TF-GRPO | verifier；无 GT 时退化为自我共识 | 去 GT 掉分（82.7→80.7）；组内共识会集体收敛到同一个错误答案 |
| ReasoningBank | **同底座自我裁判**（temp=0 二值） | 附录自认"test 时无 GT 只能自判"；误判永久写入记忆、无审计无退场 |
| AWM online | 单个冻结二值 judge | 原文自认"误判轨迹会诱导出错误 workflow 并级联" |
| EvoLM / 共进化系 | rubric 与策略共进化 | 原文自认"缺外部锚定，可能共同漂向伪指标" |
| SkillOpt | held-out split 严格 > 门 | 依赖 verifier；无置信控制；开放域不可用 |
| GRASP | 原始计数门（F−R>0） | 无统计控制、无校准、纯 on-policy |

**结论**：这个领域的 reward 信号要么靠 verifier（开放域不存在），要么靠自我裁判/自我共识（循环污染），准入全是点估计，**没有一篇对"优化器正在攻击裁判"做任何防御或测量**。

## 二、七大设计缺陷（63 篇文献交叉确认）

| # | 缺陷 | 关键证据 |
|---|---|---|
| **D1** | **自我裁判循环性**：裁判与被优化 agent 同底座/同家族，继承同样盲区 | 自偏好偏置 10–20pp 且只有 40–60% 可消除；judge 弱于 generator 达 −14.2pp；ReasoningBank/TF-GRPO/EvoLM 全中 |
| **D2** | **静裁判对动分布**：优化器每轮故意移动轨迹分布，裁判一次性校准后冻结；conformal 保证因交换性破坏而失效 | 所有平台（MLflow/LangSmith/Vertex）只做一次性对齐；唯一共进化的 RubricARM 需要 RL+标签 |
| **D3** | **点估计准入、无统计控制、无弃权**：数千候选对着"严格>"小验证集 → 靠噪声中选是必然 | SkillOpt/GEPA/GRASP/MetaSkill-Evolve 全中；近阈值噪声与真实提升不可分 |
| **D4** | **Goodhart 不可测也不设防**：meta-benchmark 只测静态人类一致率，不测"抗攻性"；平均指标掩盖局部 exploit；rubric 可被优化器带松 | GenRM scaling law：静态精度排名在优化压力下会反转；2602.13576 漂移攻击无人防 |
| **D5** | **轨迹失明**：二值结果分掩盖部分作弊；step-PRM 有正标签偏置且惩罚"犯错后恢复"；过程/结果分歧从不用作否决信号；轨迹里的确定性证据（工具报错、DB 状态差分）没人利用 | τ²-bench 上最强 judge AUROC<0.65（2606.09863）；生产环境操作性召回 0%（2606.10315）；step 一致率仅 67–72% |
| **D6** | **不可逆污染**：误判轨迹变成永久记忆/skill，无追溯审计、无退场机制 | ReasoningBank/AWM/Memento 全中，单向污染循环 |
| **D7** | **成本结构失配**：每轮数千次调用全走均价前沿模型；鲁棒性手段（集成/思考型RM/辩论）成本翻倍但抗攻性并不随之提升（同家族集成盲区相关） | SkillCoach 每轨迹一次 Gemini-Pro；级联只存在于工程博客的口口相传 |

## 三、六个改进方案（每个对应缺陷，可独立成模块）

### P1 证据优先的轨迹裁判（治 D5、D7，兼防 D4）
裁决顺序反转：**先跑确定性证据层**——工具调用 schema 合法性、错误码、DB 状态差分、里程碑断言，全部程序化提取（零 LLM 成本、不可被文本讨好）；LLM 只裁决确定性层无法判定的剩余部分，且必须引用证据。**恢复感知记分**：后续步骤修复了早期错误要给分（现有 step-PRM 一律罚，导致压制探索）。过程分与结果分**分歧时触发否决/升级而非平均**（内置 hacking 检测器——文献确认无人做）。

### P2 校准弃权 + 升级路由（治 D3、D2）
裁决输出三值：**过 / 不过 / 弃权**，置信度用**漂移条件化 conformal**（每轮用当前策略分布重估阈值——现有 conformal judge 全是一次性校准，保证在优化压力下静默失效）。弃权不是终点而是路由：送异质小组/工具探针/人工。**弃权即探索信号**：优化器推向的新区域恰是裁判最不确定的地方，把这些样本送贵验证而不是丢弃（现有选择性预测裁判把弃权当否决，恰好在探索前沿掐断信号）。

### P3 异质防串谋小组 + 分歧告警（治 D1、D4）
小组成员**按盲区不相交工程化设计**：不同模型家族、不同输入框架（含作者身份盲化——自归因偏置由框架而非标签驱动）、不同 rubric 分解。**分歧不平均、当告警**：成员分歧超阈值 = 疑似 hacking，触发否决或升级（文献确认"分歧有诊断力但无人在环内用它做门"）。加**轮换隐藏裁判**：优化器只见在环裁判，隐藏成员随机抽样参与裁决，Goodhart 压力无法拟合它看不到的目标。

### P4 裁判自身的 CI/CD（治 D2、D4、rubric 被带松）
把裁判（prompt+模型+few-shot+rubric）当**版本钉死的工件**：每次裁判更新必须通过自己的回归套件——冻结金锚集一致率 + **对抗 exploit 集**（历史上骗过裁判的样本库）。**秘密轮换金丝雀**：每 N 轮用优化器永远看不到的隐藏金集重打分，"headline reward 上涨而金丝雀一致率横盘/下跌" = Goodhart 警报 → 自动冻结/回滚。这直接把 2602.13576 的漂移攻击从"离线研究"变成"环内防御"。

### P5 配对反事实裁决 + 统计准入（治 D3、D2）
准入裁决不打绝对分，**只裁配对差**：同输入下 with-entry vs without-entry 轨迹对比（绝对刻度漂移被差分消掉）。准入门用统计下界 + 多重比较控制（sequential testing / FDR——数千候选场景下"验证噪声中选者"被构造性拒绝）。**与 REPLAY 的定向微验证天然同构**——`shared/stats.py` 已就位。

### P6 成本级联 + 迭代调度（治 D7、D2）
四层级联：确定性证据层（免费，100%）→ 廉价筛裁判（haiku 级，100%）→ 异质小组（仅近阈值/弃权/分歧样本）→ 前沿审计员（分层抽样复核）。**廉价层与审计层的分差本身就是漂移统计量**，超阈值触发重校准（工程博客folklore，从未被产品化/研究化）。**预算按迭代调度**：越到后期策略被优化得越狠、gaming 风险越高，验证预算随迭代后移（现有成本优化全是单一静态工作点）。

## 四、最优合并设计：**GATEKEEPER**（工作代号）——self-evolve 专用裁判栈

```
                    ┌─ 每 N 轮: 秘密金丝雀重打分 + 锚集回归 + exploit 集 (P4)
   候选轨迹/条目 ──▶ L0 确定性证据层 (P1)          ──┐ 免费, 100% 覆盖
                    L1 廉价筛裁判+校准弃权 (P2,P6)  ──┤ ~$0.001/条, 100%
                    L2 异质小组, 分歧=告警 (P3)     ──┤ 仅弃权/近阈值/分歧样本
                    L3 配对反事实 + 统计准入门 (P5) ──┘ 仅准入决策
   输出: 过/不过/弃权+升级  + 每 rubric 项的结构化不确定性反馈(优化器可读)
   元层: 版本钉死+裁判CI (P4) · 漂移条件化重校准 (P2) · 追溯审计退场 (D6) · 迭代调度预算 (P6)
```

设计原则一句话：**便宜的层永远先跑、确定性的证据永远压过文本印象、分歧永远是信号而非噪声、优化器永远面对一个它无法完全看见且持续被审计的目标。**

### 为什么这是合并而不是堆料
每层治的缺陷不同且互为前提：没有 L0 的证据锚定，L1-L3 全是可被讨好的文本印象（D5→D4）；没有 P2 的弃权，L2 不知道该看哪些样本（D3→D7 成本爆炸）；没有 P4 的元层，整个栈会随策略一起漂移（D2）。去掉任何一层都有对应的、已被文献量化的失效模式——这本身就是论文的消融表。

### 评估方案（成本极低，主要是单次调用）
1. **静态质量**：τ²-bench 轨迹上对照隐藏 verifier 的 AUROC——对标 2606.09863 的 <0.65 天花板（我们手上 170 条带真值轨迹零成本起步，pilot 单层 judge=0.68 已是不错起点）；
2. **抗攻性（新指标，文献确认无人做）**：专门的 red-team 文本优化器攻击各裁判配置，测 reward-gap 增长率——"hackability 作为裁判的第一公民指标"；
3. **环内端到端**：接入 skill 优化循环（REPLAY 管线现成）做准入门，对照 naive judge / verifier oracle 测**假准入率/漏准入率**；
4. **成本-质量 Pareto**：级联 vs 全前沿裁判。
