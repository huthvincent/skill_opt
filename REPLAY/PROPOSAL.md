## 0. 一段话摘要

Training-free self-evolve（skill/经验库优化）的 reward 信号是 LLM 裁判，而裁判在优化压力下必被攻破（七缺陷分析见 `../docs/judge_design_20260706.md`）；"裁判必须共进化"已成共识（Red Queen Gödel Machine 等），但**没有任何工作知道共进化该停在哪**——没有终点的军备竞赛无法部署。CAGE 给出第一个**锚定的终止证书**：agent（生成器）、裁判栈（判别器）与显式红队（作弊者）三方对抗共进化，全程被一组双方不可触碰的锚（冻结金锚集+秘密金丝雀+确定性证据层）"关在笼中"；当**三条件认证均衡**成立——真实进步边际趋零（C1）、红队定额攻击产率灭绝（C2）、裁判锚上精度不降（C3）——即宣告收敛。单值判据为**欺骗溢价 DP**（骗过门的边际成本 ÷ 真做对的边际成本）持续 >1：GAN 收敛于"以假乱真"，CAGE 收敛于"作假不如做真"。

## 1. 问题设定

- 冻结 LLM agent 策略 π(y|q,E)，E=文本经验库；优化器从交互日志提出条目、经裁判门控准入（= v1 REPLAY 管线，完整保留为 G 回合）。
- 无 ground-truth reward；可用：LLM 裁判（已知领域天花板 AUROC<0.65，judgeauroc2026）+ **稀疏标注审计流**（少量带真值的锚样本；实验中由隐藏 verifier 扮演，仅测量用，永不进优化）。
- 威胁模型：优化器天然 Goodhart 裁判；显式红队将此压力具象化、可测量化。

## 2. 方法：CAGE 协议

### 2.1 三回合循环
1. **G 回合（agent 进化）**：v1 管线——日志失败聚簇 → 单簇 3 候选 → 簇匹配定向配对验证（裁判信号）→ 统计悲观准入（`shared/stats.py`，LCB>0 + 无害对照）→ 条件注入。
2. **A 回合（攻击）**：红队对当前版本裁判做黑盒定额攻击（预算 B_att/轮）：(a) 轨迹级——把失败轨迹改写成"假成功"（伪造工具结果转述、无凭断言解决、冗长讨好、格式伪装）；(b) 条目级——提交"作弊条目"（教 agent 声称成功而非做成）走 G 回合的门。**红队产物天然自标注**（它知道自己是假的）——骗过门的进作弊档案，无需 verifier。
3. **D 回合（裁判进化）**：从作弊档案打补丁（反例 few-shot / rubric 收紧 / L0 新增确定性探针如"声称成功但无对应工具调用"）；**补丁必须过裁判 CI**——冻结金锚集精度不降（anytime-valid 序贯检验，机器借 2606.15474）且历史档案捕获率达标——才准上岗，否则回滚。

### 2.2 判别器架构 = GATEKEEPER（`../docs/judge_design_20260706.md`）
L0 确定性证据层（工具 schema/错误码/声称-行为一致性——只用轨迹内证据，**不含**隐藏 verifier）→ L1 廉价筛+校准三值输出（过/不过/弃权）→ L2 异质小组、分歧=告警→升级 → L3 配对反事实+统计门。元层：版本钉死+CI、秘密金丝雀、追溯审计。

### 2.3 锚（笼子）——三类，均不参与进化
- **冻结金锚集**：设置期一次性标注（实验=隐藏 verifier；部署=人工抽标），裁判 CI 的考卷；
- **秘密金丝雀**：优化器与红队全程不可见的锚子集，只用于向实验方报告（防锚被过拟合）；
- **确定性证据层 L0**：文本讨好不可动摇的"物理定律"。

### 2.4 认证终点

| 条件 | 判据 | 排除的假收敛 |
|---|---|---|
| C1 | 锚上真实成功的边际提升/预算 < δ，持续 k 轮 | 过早收工 |
| C2 | 红队定额 B_att 的骗成产率 < ε，持续 k 轮（=裁判经验 exploitability≈0） | 潜伏漏洞（主动认证） |
| C3 | 裁判在金锚集+金丝雀+历史档案上的精度 ≥τ 且不降 | 双双漂移/裁判被打残 |

**欺骗溢价 DP** = (B_att ÷ 骗成数) ÷ (G 回合验证预算 ÷ 真准入数)。终点 ⇔ C1∧C2∧C3 持续 k 轮 ⇔ DP>1 持续。新颖性锚定：该判据未被使用（种子引 2510.01367、2606.26071 为非实例）；C2 形式化=预算化经验 exploitability 施于裁判。

### 2.5 信息卫生（实验有效性的命门）
| 角色 | 可见 | 不可见 |
|---|---|---|
| agent/优化器 | 裁判裁决 | verifier、锚、金丝雀、档案 |
| 红队 | 裁判裁决（黑盒） | verifier、锚、金丝雀 |
| 裁判补丁器 | 作弊档案（自标注）、金锚集（自己的 CI） | 金丝雀、verifier 在线信号 |
| 测量 harness | 一切（verifier 只在此层） | — |

## 3. 实验

### 3.1 MVP（另一台 server / Bedrock 执行，指令文档另发 GAA.git，预算帽 $1000）
retail 单域、≤8 轮、三臂对照：**CAGE** vs **B1 静态裁判**（无 D 回合——领域默认姿势，预期：作弊产率居高、假条目入库、judge 分与真实分背离扩大）vs **B2 无证书共进化**（D 回合无 CI 门——RQGM 式，预期：锚上精度漂移）。招牌图：三臂的 DP 曲线 + 背离曲线 + 锚精度曲线。详细设计与输出规范见指令文档。

### 3.2 正式实验（MVP 绿灯后）
+airline 域与 WildChat 设定；证书消融（去 C1/C2/C3 各自复现对应假收敛）；红队强度扫描（攻击预算-产率全曲线）；对照攻击库 2503.04474；hackability 作为裁判选型指标 vs 静态一致率选型的对比。预注册规则沿用 `../docs/review_response_20260706.md` §四。

## 4. 贡献清单

1. **CAGE 协议**：首个 training-free、锚定的 agent–裁判对抗共进化（含显式红队与自标注作弊档案闭环）；
2. **认证终点**：三条件均衡 + 欺骗溢价——第一个把"作弊经济学"变成收敛判据的工作；
3. **GATEKEEPER 判别器架构**（七缺陷驱动的工程设计）；
4. **实证**：静态裁判被攻破的量化、DP 曲线、假收敛复现。

## 5. 风险与对策

| 风险 | 对策 |
|---|---|
| RQGM/CoEvoSkills 补上证书（都是 6 月的活跃工作） | MVP 出 DP 曲线立即挂 arXiv 占位 |
| 红队太弱 → C2 假达标 | 报告攻击预算-产率曲线而非单点；攻击族多样性硬性要求；引入 2503.04474 攻击库保底 |
