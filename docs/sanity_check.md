# REPLAY Sanity Check v2 —— 验证"重设计后的方法到底 work 不 work"

> **v2 修订（2026-07-05 晚）**：v1 的 Gate 1（批量生成 12 条整库注入测天花板）**设计本身就是错的**——它测的是方法明确禁止的姿势，三次尝试花 $18.4 无干净判定但趋势与 Skill-RM 消融一致（裸堆知识有害）。v2 按重设计后的方法（`../REPLAY/PROPOSAL.md` v1：逐条贪心准入 + 定向微验证）重排门序。

> 判读纪律（来自 v1 的实测教训）：**n=60 全集噪声带 ±10pp**——一切判定基于配对比较和定向集大效应，不看全集小差值。

## 可复用家底（省钱清单）

- 干净基线：haiku 无库 retail-30 = **70%**（n=60），12 个失败任务 id 已知 → **直接当作"日志"用，失败簇现成**；
- 12 条天花板库（`gate1_ceiling_20260705_190509/library.md`）→ 候选条目的现成起点，省提案费；
- ~170 条带隐藏 DB 真值的轨迹 → G2 的评测集，零生成成本。

## G0 —— 可控性预门：agent 到底"教得动"吗？（~$2）

**回答**：Gate 1 失败的备择解释——也许 haiku 连"完美针对性指令"都执行不了。不排除这个，一切条目优化都是空谈。
**做法**：取 3 个基线稳定失败的任务（如 task 3/6/7，基线 0%），每个由强模型**看着标准答案**写一条"金条目"（允许作弊到底、允许任务专属），单任务注入单条目，各跑 2 trial。
**判定**：≥2/3 任务翻案 → 过（条目能改变行为，问题只在条目质量与注入方式）；<2/3 → **停**：先换 sonnet 当 agent 重测 G0（+$1），仍不过则该域对指令注入不敏感，回来重议。

## G1 —— 单条目全管线：方法的最小完整闭环（~$5）

**回答**：核心问题——**一条从日志挖出、经定向验证的条目，能不能产生真实、可测的收益？** 这是 v2 的心脏，等价于旧 Gate 1 的"天花板"但姿势正确。
**做法**（完整走一遍 PROPOSAL v1 的步骤 1–3）：
1. 从现有基线失败轨迹（=日志）聚出 top-1 失败簇（已知 12 个失败任务里挑同模式的 8 个）；
2. optimizer 写 3 个候选变体（或直接从 12 条旧库里挑 3 条匹配该簇的——更省）；
3. 每变体配对 A/B：定向集 8 任务 × 2 trial × 2 臂 + 无害对照 5 任务 × 1 trial；
4. 准入判据：定向集 Δ 的 90% 置信下界 > 0 且对照不掉分。
**判定**：≥1 个变体通过准入门 → **过**（方法闭环成立！）；0 个通过但定向集有正向趋势 → 黄灯，看是变体质量还是门太严；全部无效乃至负向 → **停**，结合 G0 结果重议（可能该域 policy 已足够好，需换更难的域如 telecom）。

## G2 —— 便宜信号检查：judge+隐式信号准不准？（~$4）

**回答**：日志挖掘那半的地基（同 v1 的 Gate 2）。真实设定里没有 verifier，判"哪些 session 失败了"靠 judge+隐式信号——它得跟真相相关。
**做法**：拿现有 ~170 条带真值轨迹，judge（haiku，看不到 verifier）+ 隐式信号给每条打分，对照隐藏 DB 真值算 AUC。**零生成成本，只有判卷费**。
**判定**：AUC ≥ 0.75 → 过；0.6–0.75 → 黄灯（上多 judge 集成再测，+$2）；≈0.5 → **停**（离线挖掘地基不成立）。

## G3 —— 核心主张：同预算打赢纯 on-policy（~$40）

**回答**：论文成败——同样 rollout 预算下，"日志定向"是否优于"on-policy 自己探索"（SkillOpt/TF-GRPO 姿势）。
**做法**：预算 B≈150 集，两臂各自完整优化：(a) 纯 on-policy：探索+验证全靠现场跑；(b) REPLAY v1：失败簇免费来自日志，预算全花在定向验证。终评 held-out 30 任务 × 2 trial 配对比较（McNemar）。
**判定**：(b) 准入条目更多/更优或同等成果省 ≥1/3 预算 → **绿灯，申请主实验**；平手 → 黄灯（靠样本效率曲线叙事）；(b) 更差 → 停，重议。

## G4 —— 招牌图：judge-only 优化会跑偏（~$12，选做）

同 v1 的 Gate 4：验证门换成纯 judge 跑 G1 管线，隐藏 verifier 盯着画"judge 分涨 vs 真实分背离"，再叠加双信号锚版本展示收窄。不毙论文，出图进 intro。
